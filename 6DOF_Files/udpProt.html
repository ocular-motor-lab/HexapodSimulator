3. Analysis of the action command UDP data area
1. Verification code
Fixed value, 0x55aa, hexadecimal, 2 bytes.
2. Pass code
Fixed value, 0x0000, hexadecimal, 2 bytes.
3. Function code
Divided into two types: 0x1301, 0x1401, hexadecimal, and 2 bytes.
The absolute time playing function code is 0x1301, and the relative playing time function code is 0x1401.
The absolute time 0x1301 matches the movie. When the absolute time is played, the execution time of the command is the absolute time in the current command.
Subtract the absolute time in the last instruction from the time. For example, the absolute time of your last instruction is 15000ms, now send
This one is 15100ms, then the execution time of this instruction is now 100ms (15100ms-1500ms).
The relative time playback function code 0x1401 is the Delta Time in the current command, which is the execution time of the current command.
MBOX calculates the speed of the motor based on this execution time.
If the function code 1301 is played in absolute time and the time code is 0, then the play time of each frame of data will be based on the internal time.
Play time (default 100ms); if the relative time function code is used, the time code is 0, then the play time of each frame of data is also
It will be played according to internal time (default 100ms).
But it is recommended not to set the time code to 0, please set it according to the actual execution time.
4. Channel number
Hexadecimal, 2 bytes.
Make sure that the current command is 3 axis, 6 axis, 10 axis control command, 3 axis command is 0x0000, 6 axis command is 0x0001,
The 10-axis command is 0x0002.
5. Who accepts and who responds
Hexadecimal, 2 Bytes each, 4 Bytes in total.
Each MBOX has an IP address, the factory default is 192.168.15.201 (can be modified, please refer to MBOX for modification method)
Manual), and supports responding to each control command.
For example, if the sent instruction wants the IP address of 192.168.15.201 to receive and reply, whoever receives it is 0x0fc9 (0x0f=15,
0xc9=201), who replies to 0x0fc9 (0x0f=15, 0xc9=201). If you control multiple platforms at the same time, hope that each platform
All can be received, and each platform replies, and whoever receives it is 0xffff, and whoever replies is 0xffff. Set who replies to 0x0000,
Then the platform does not respond to the received instruction; whoever responds is set to a certain fixed value, then the corresponding one will respond (such as who responds
If it is set to 0x0510, the corresponding IP address is followed by 192.168.5.16 for command response). For rows and columns to receive replies in detail
See MBOX manual.

8. Location information
Hexadecimal, 4 Bytes for each position information.
The unit is the number of pulses.
How much position information is determined by the 4th channel number, if the channel number is 3 axis command 0x0000, some 3
The position information is X, Y, Z (corresponding to axis 1-3), a total of 12 bytes; if the channel number is 6-axis command 0x0001,
Need some 6 position information respectively X, Y, Z, U, V, W (corresponding to 1-6 axis), a total of 24 Byte;
The channel number is the 10-axis command 0x0002, and some 10 position information is needed for X, Y, Z, U, V, W, A, B, C,
D (corresponding to axis 1-10), a total of 40 bytes.
Pulse calculation method: the factory defaults that the number of pulses for one motor revolution is 10000
(1) Examples of electric cylinders:
Suppose, the lead of the electric cylinder is 5 mm, and the number of pulses per revolution is 10,000. When the motor rotates once, the electric cylinder rises by one lead.
Electric cylinders are from 0 to 25 mm, calculate the number of pulses required: 25/5*10000=50000, converted into hexadecimal is
0x0000c350, (Note: The length of each cylinder can be set as required.)
(2) Examples of rotation axis:
The rotating shaft generally has a deceleration. If the deceleration ratio is 1:60, the motor makes 60 revolutions and the turntable makes one revolution. If turn
The table needs to rotate 30 degrees, calculate the required number of pulses 30/360*(60*10000)=50000, and the conversion to hexadecimal is
0x0000c350.

3. Function code

4. Channel
Hexadecimal, 2 bytes.
Determined by the 3rd function code.
If the function code reads register 0x1101, the channel is 0x0000 to read DN parameters, and the channel is 0x0001 to read FN parameters.
For example, if the function code is written to register 0x1201, the channel is 0x0000 and the FN parameter is written but not saved, and the channel is 0x0001.
FN parameters and save, channel 0x0002 write CX control parameters.

Hexadecimal, 2 bytes.
It supports single reading and continuous multiple reading, so it is the starting address of the parameter to be read.
7. Read the number of registers
Hexadecimal, 2 bytes.
How many bytes are read consecutively, including the start address.
8. Parameter value
If it is a read parameter command, the command has no parameter value part. If it is a write parameter, it is the value of the register.
